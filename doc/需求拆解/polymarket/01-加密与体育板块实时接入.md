# Polymarket 加密与体育板块实时接入

## 背景

本次是 Polymarket 调研和对接第一版，只接公开只读数据，不落库、不下单、不接钱包签名。

展示目标是快速查看 Polymarket 加密货币板块和体育板块的活跃事件，以及事件下的市场数据。

## 官方接口

Gamma API Base URL：

```text
https://gamma-api.polymarket.com
```

加密货币板块事件：

```bash
curl "https://gamma-api.polymarket.com/events?tag_id=21&active=true&closed=false&limit=20&order=volume24hr&ascending=false"
```

体育板块事件：

```bash
curl "https://gamma-api.polymarket.com/events?tag_id=1&active=true&closed=false&limit=20&order=volume24hr&ascending=false"
```

按市场 slug 查询：

```bash
curl "https://gamma-api.polymarket.com/markets/slug/btc-updown-5m-1778746200"
```

板块 tag 映射：

```text
crypto -> tag_id=21
sports -> tag_id=1
```

## 本地接口

本地后端在 `xms-agent-app` 提供实时代理接口：

```text
GET /api/polymarket/events?section=crypto|sports&limit=20&offset=0
GET /api/polymarket/crypto/events
GET /api/polymarket/crypto/updown?coins=btc,eth,sol&before=2&after=6
GET /api/polymarket/sports/events
GET /api/polymarket/markets/slug/{slug}
```

返回数据保留 Polymarket 原始字段，并补充调试字段：

```text
section
tagId
sourceUrl
fetchedAt
limit
offset
count
events / market
```

## 独立页面

调试页面：

```text
http://localhost:18675/polymarket.html
```

页面能力：

- Crypto / Sports 切换。
- 修改 limit 后刷新事件。
- 展示事件 title、slug、volume24hr、liquidity、endDate、markets 数量。
- 展示前几个市场的 question、outcomePrices、bestBid、bestAsk、lastTradePrice。
- 输入 market slug 查询市场详情原始 JSON。

## 边界

- 第一版只走实时代理，不写数据库。
- 不做定时同步、不做缓存策略。
- 不做 CLOB 下单、撤单、钱包、签名和额度授权。
- 如果后续要做交易或行情归档，需要另开二期设计。
