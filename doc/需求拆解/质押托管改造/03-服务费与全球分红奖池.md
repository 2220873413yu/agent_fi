# 第三批：服务费与全球分红奖池

## 目标

在静态收益发放时按托管周期扣除服务费，并将扣出的分红服务费记录到全球分红奖池。

## 已知规则

- 服务费比例后台可配置。
- 服务费内扣。
- 服务费比例按托管周期不同而不同，不是统一比例。
- 扣除的服务费作为分红服务费进入全球分红奖池。
- 后台可设置增减全球分红奖池。

当前已确认的托管周期服务费比例：

| 托管周期 | 服务费 |
| --- | ---: |
| 1天 | 50% |
| 30天 | 30% |
| 90天 | 20% |
| 180天 | 10% |
| 360天 | 5% |

## 奖池资金来源与用途

全球分红奖池只记录可用于每周全球分红的 USDT 金额，不直接代表用户资产。

奖池金额来源：

- 每天 101 托管静态收益任务按托管周期扣除的分红服务费。
- 后台人工手动增加奖池金额。

奖池金额支出：

- 每周日全球分红结算时，从奖池扣除本次实际用于发放的分红金额。
- 后台人工手动扣减奖池金额。

101 定时任务处理单笔托管订单时，计算顺序为：

1. 计算静态毛收益。
2. 如订单存在已生效、未退还 AFI 加速记录，则静态毛收益乘以质押记录中的历史快照倍率。
3. 按套餐服务费比例计算服务费。
4. 用户到账静态收益 = 静态毛收益 - 服务费。
5. 全球分红奖池入池金额 = 本次扣除的服务费。
6. 写入奖池余额，并写入奖池流水。

当前已确认默认口径：

- `50%` 是 1 天托管周期的服务费比例，不是所有服务费再按 50% 入池。
- 不同托管周期按套餐配置读取不同服务费比例。
- 本阶段按本次扣除的服务费全额入全球分红奖池处理。
- 每天服务费入池必须有流水，按当天服务费累计金额写入一条流水，方便追溯这笔钱来自哪个结算日。

## 奖池表设计口径

全球分红奖池使用一张余额表，原则上只保留一条有效记录。

建议记录字段：

- 奖池 ID。
- 当前奖池余额。
- 累计入池金额。
- 累计支出金额。
- 最近入池时间。
- 最近支出时间。
- 备注。

该表只负责保存当前奖池状态，不保存每一笔资金变化原因。

## 奖池流水表设计口径

奖池流水表记录奖池每一笔增加和扣减，用于审计和后台查询。

建议记录字段：

- 流水 ID。
- 流水单号。
- 流水类型：收入、支出。
- 业务类型：
  - 每日服务费入池。
  - 后台手动增加。
  - 每周全球分红扣减。
  - 后台手动扣减。
- 变动金额。
- 变动前奖池余额。
- 变动后奖池余额。
- 来源订单 ID / 订单号。
- 来源用户 ID。
- 来源结算日。
- 来源批次号。
- 操作人。
- 备注。

每日服务费入池时，不按每条托管订单写流水，只按当天累计服务费写一条收入流水，流水记录来源结算日。

每周全球分红扣减时，流水需要能追溯到全球分红结算批次。

## 待确认问题

- 服务费比例是全局配置，还是按 G 值区间/套餐配置。
- 服务费比例后续是否只按托管周期配置，还是还会叠加 G 值区间/套餐其他规则。
- 平台沉淀是否需要流水表。
- 奖池调增调减是否需要审核。
- 服务费扣除后团队奖励按净收益还是毛收益，文档倾向净收益。

## 表结构影响

可能需要新增：

- 全球分红奖池余额表，只保留一条有效奖池记录。
- 全球分红奖池流水表，记录每日服务费入池、后台调账、每周分红扣减。

## 后端影响

- 静态收益发放拆分为：
  - 理论收益。
  - 服务费。
  - 用户实际到账。
  - 奖池入账。
  - 平台沉淀。
- 101 任务每天发放静态收益时，需要累计当天所有托管订单服务费，并在任务完成后写入一条奖池收入流水。
- 101 任务服务费比例需要按托管订单对应套餐周期读取，例如 1天 50%、30天 30%、90天 20%、180天 10%、360天 5%。
- 后台需要支持手动增加、扣减奖池金额，并同步写入奖池流水。
- 每周全球分红任务发放时，需要从奖池扣减实际分红金额，并写入支出流水。

## 验收用例

- 收益内扣服务费正确。
- 用户到账为净收益。
- 不同托管周期按不同服务费比例扣费。
- 本次扣除的服务费写入全球分红奖池。
- 后台手动增减奖池有流水。
- 每日服务费入池流水按结算日汇总，一天一条，不按每条托管订单生成。
- 每周全球分红支出流水可追溯到分红批次。
