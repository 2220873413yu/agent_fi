# 第六批：全球分红与后台管理

## 目标

实现每周全球分红结算和相关后台管理能力。

## 已知规则

- 全球分红每周日晚上 24 点结算。
- 所有同等级节点，按照每周新增业绩加权分。
- 后台可调比例。
- 全球分红资金来源于服务费奖池。

## 分红金额来源

全球分红每周可分配金额来自全球分红奖池。

奖池收入包括：

- 101 托管静态收益任务每天按托管周期实际扣除的分红服务费入池金额。
- 后台手动增加的奖池金额。

托管周期对应的服务费比例按套餐配置读取，例如 1天 50%、30天 30%、90天 20%、180天 10%、360天 5%。这里的 `50%` 是 1 天周期服务费比例，不是所有服务费再统一按 50% 入池。

奖池支出包括：

- 每周日全球分红实际发放金额。
- 后台手动扣减的奖池金额。

每周分红任务执行时，需要基于奖池当前余额确定本周可发放金额。实际发放后，需要从奖池余额中扣除，并写奖池流水。

## 每周分红分配口径

每周全球分红按“同等级节点的本周新增小区业绩”加权分配。

本周新增小区业绩口径以 `08-每周新增小区业绩.md` 和 `10-新增小区业绩定义.md` 为准：

```text
本周新增小区业绩 = 本周末小区有效积分快照 - 上周末小区有效积分快照
```

托管订单参与计算时使用托管积分：

```text
托管积分 = 托管金额 × 套餐积分系数
```

已确认口径：

- 参与分红的等级使用 F 等级配置。
- 各 F 等级的分红比例使用 `t_user_level_config.global_fee_dividend_ratio`。
- 同一个全球分红奖池先按 F1-F9 配置比例拆分成各等级奖池。
- 每个等级内部，按 `community_new_performance > 0` 的用户加权分配。
- `community_new_performance <= 0` 的用户只保留周业绩记录，不参与同等级分母，也不生成分红明细。
- 如果某等级正数小区新增业绩总和为 0，则该等级本周暂不发放，未发放金额留在奖池。

示例：

```text
本周全球分红可分配金额 = 10,000 USDT
F5 全球手续费分红比例 = 8%
F5 本周等级奖池 = 10,000 × 8% = 800 USDT

F5 用户 A 本周新增小区业绩 = 20,000
F5 用户 B 本周新增小区业绩 = 60,000
F5 本周新增小区业绩总和 = 80,000

A 可得 = 800 × 20,000 / 80,000 = 200 USDT
B 可得 = 800 × 60,000 / 80,000 = 600 USDT
```

非正数示例：

```text
同等级用户 A 本周新增小区业绩 = -800
同等级用户 B 本周新增小区业绩 = 0
同等级用户 C 本周新增小区业绩 = 4,700
同等级用户 D 本周新增小区业绩 = 3,000

同等级分红总权重 = 4,700 + 3,000 = 7,700
```

A 和 B 不参与分红，也不生成全球分红明细。

## 奖池扣减口径

每周全球分红任务完成发放后，按实际发放总额扣减全球分红奖池。

需要写入：

- 全球分红结算批次。
- 全球分红发放明细。
- 用户 USDT 钱包流水。
- 奖励记录。
- 奖池支出流水。

奖池支出流水需要记录：

- 分红批次号。
- 本次计划分红金额。
- 本次实际发放金额。
- 扣减前奖池余额。
- 扣减后奖池余额。

如果存在未分完金额，本阶段先不确定处理方式，后续业务确认后再定。

## 待确认问题

## 表结构影响

可能需要新增：

- 全球分红奖池余额表。
- 全球分红奖池流水表。
- 全球分红配置表。
- 每周新增小区业绩快照表。
- 全球分红结算批次表。
- 全球分红明细表。

## 后端影响

- 新增每周任务。
- 按等级和每周新增小区业绩权重分配。
- 每周分红前读取全球分红奖池余额。
- 每周分红后扣减奖池，并写奖池支出流水。
- 发放 USDT 并写奖励记录。
- 后台支持查询、导出、补发、重算。
- 后台支持奖池余额查询和手动调增调减。

## 验收用例

- 每周只结算一次。
- 每周分红金额来自全球分红奖池。
- 同等级用户按本周新增小区业绩权重分配。
- 后台调比例生效。
- 后台手动增加、扣减奖池后，奖池余额和流水正确。
- 每周发放后，奖池余额按实际发放金额扣减。
- 重复任务不重复发放。
