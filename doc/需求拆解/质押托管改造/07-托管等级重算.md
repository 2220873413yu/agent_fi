# 托管等级重算

## 目标

从质押托管订单支付成功开始，异步重算用户托管业绩、小区业绩和真实等级，为后续团队奖励发放提供统一等级口径。

## 触发时机

- 用户链上支付回调成功后，托管订单进入 `产出中`，开始计入等级考核。
- 后台拨付托管订单创建成功后，同样进入 `产出中`，开始计入等级考核。
- 待支付订单不参与业绩和等级考核。
- 订单完成并扣减托管业绩后，需要重新计算相关用户等级，允许真实等级下降。

## 异步处理

- 复用现有 Redis Stream：`ORDER_DYNAMIC_SETTLEMENT`。
- 新增托管等级重算业务类型：`OrderMsgDO.bizType = 4`。
- `bizType=4` 只处理托管等级重算，不复用买节点 `bizType=1` 的节点奖励逻辑。
- 发送消息建议放在订单支付/拨付事务提交后，避免事务回滚但消息已发送。

## 等级字段口径

`t_user_info` 使用三个等级字段：

- `game_level`：真实等级，由系统按当前个人业绩和小区业绩计算。
- `min_game_level`：赠送等级，由买节点逻辑写入，后台只展示，不允许手动编辑。
- `admin_game_level`：管理员保底等级，新增字段，后台用户管理可展示和编辑。

奖励发放、上级链路结算、实际等级展示时统一取：

```text
实际奖励等级 = max(game_level, min_game_level, admin_game_level)
```

## 等级考核口径

本批只考核：

- 个人托管业绩：`t_user_info.performance`
- 小区托管业绩：`t_user_info.community_performance`

不考核：

- 大区业绩：历史字段，当前需求不使用。
- 线数条件：`required_leg_num`、`leg_level_min`、`leg_level_count` 保留但废弃，代码不参与判断，后续表注释需要标明废弃。

小区业绩计算复用现有代码逻辑：

```text
每条直推线业绩 = 直推用户个人托管业绩 + 直推用户伞下团队托管业绩
小区业绩 = 所有直推线业绩之和 - 最大直推线业绩
```

## 等级配置

复用 `t_user_level_config`：

- `level`：F 等级，`1=F1`，`2=F2`，直到 `9=F9`。
- `performance`：个人托管业绩要求。
- `community_performance`：小区托管业绩要求。
- `team_performance`：历史大区字段，本需求不参与考核，初始化为 `0`。
- `required_leg_num`、`leg_level_min`、`leg_level_count`：废弃字段，初始化为 `0`。

F1-F9 初始化口径：

| 等级 | 个人托管 | 小区托管之和 |
| --- | ---: | ---: |
| F1 | 500 | 5000 |
| F2 | 1000 | 20000 |
| F3 | 1000 | 30000 |
| F4 | 2000 | 100000 |
| F5 | 2000 | 350000 |
| F6 | 2000 | 700000 |
| F7 | 3000 | 1500000 |
| F8 | 5000 | 3000000 |
| F9 | 5000 | 6000000 |

## 处理流程

1. 托管订单支付成功或后台拨付成功。
2. 增加当前用户个人托管业绩。
3. 增加直推上级直推托管业绩。
4. 增加所有上级伞下团队托管业绩。
5. 事务提交后发送 Redis 消息：`bizType=4`。
6. 消费端按下单用户找到其上级链路。
7. 对下单用户和上级链路重新计算小区业绩。
8. 按 `t_user_level_config` 从低到高判断，取满足条件的最高等级写入 `game_level`。
9. 如果当前业绩不满足原等级，允许 `game_level` 自动降级。

## 后台管理

- 用户列表展示：
  - 真实等级：`game_level`
  - 节点赠送等级：`min_game_level`
  - 管理员保底等级：`admin_game_level`
  - 实际奖励等级：三者最大值
- 用户编辑弹窗：
  - 只允许编辑 `admin_game_level`。
  - `min_game_level` 只展示，不允许编辑。
- 等级展示继续使用 `t_user_info_game_level` 字典，不写死枚举。

- 托管订单支付/拨付成功后发送 `bizType=4`。
- Redis 消费端新增 `bizType=4` 分支。
- 恢复或重写 `callUserLevel`，只按个人业绩和小区业绩计算真实等级。
